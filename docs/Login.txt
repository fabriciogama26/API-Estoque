Tela: Login
==============

Objetivo
-------
- Autentica usuários através do Supabase Auth antes de conceder acesso a qualquer rota protegida (`ProtectedRoute`).
- Em caso de sucesso, redireciona para o caminho originalmente solicitado ou `/`.

Campos do Formulário
-----------
- `username` (e-mail, obrigatório): usado como identificador do Supabase Auth.
- `password` (senha, obrigatório).
- `remember` (caixa de seleção): atualmente somente na interface do usuário; a persistência da sessão do Supabase é gerenciada pela configuração do SDK.

Validações
-----------
- Atributos HTML básicos `required` impedem envios vazios.
- `AuthContext.login` agora:
* Rejeita quando um dos campos está ausente (`Informe usuário e senha`).
* Chama `supabase.auth.signInWithPassword`; mensagens de erro do Supabase são exibidas para o usuário.

Fluxo de Dados
---------
- Em caso de sucesso, a sessão retornada é armazenada pelo Supabase; o perfil analisado é armazenado em cache em `localStorage` (`api-estoque-auth`) para recarga rápida.
- Em caso de falha, é exibido um texto de feedback abaixo do formulário.

Notas de Segurança
--------------
- Todas as solicitações de API subsequentes dependem do token de acesso do Supabase adicionado automaticamente por `src/services/api.js`.
- Nenhum endpoint `/api/auth/login` personalizado permanece; a autenticação é totalmente gerenciada pelo Supabase.
