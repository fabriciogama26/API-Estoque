Tela: Entradas
==============

Visão geral
-----------
- Registrar reabastecimentos de EPIs e listar o histórico filtrável por material, centro de custo e período.
- O formulário e a tabela funcionam de forma idêntica em ambos os modos de dados; apenas a origem dos registros muda.

Fluxo principal
---------------
1. Ao montar, `EntradasPage` carrega materiais e entradas em paralelo via `Promise.all` (`dataClient.materiais.list` + `dataClient.entradas.list`).
2. O formulário envia `materialId`, `quantidade`, `centroCusto`, `dataEntrada` e `usuarioResponsavel` (obtido do `AuthContext`).
3. Após salvar (`dataClient.entradas.create(payload)`), o formulário é limpo e a lista é recarregada (reseta paginação).
4. A listagem suporta filtros client-side (termo) e filtros enviados à API (material, centro de custo, data inicial/final).

Campos do formulário
--------------------
- `materialId`: select obrigatório populado com os materiais ativos retornados pelo `dataClient`.
- `quantidade`: input numérico ≥ 1.
- `centroCusto`: campo texto obrigatório (aplicado direto na tabela `entradas.centro_custo`).
- `dataEntrada`: input `type="date"` obrigatório; a data é repassada como string ISO (`yyyy-mm-dd`).

Regras de validação
-------------------
- HTML garante obrigatoriedade dos campos.
- Antes de enviar, `quantidade` é convertida para `Number`. Valores inválidos resultam em erro exibido acima do formulário.
- O backend remoto verifica a existência do material e normaliza o centro de custo; o modo local aplica validações equivalentes antes de persistir no `localStorage`.

Tabela e paginação
------------------
- Paginação cliente (`TablePagination`) com página de tamanho fixo (`TABLE_PAGE_SIZE`).
- Colunas incluem material, quantidade, valor total calculado (`quantidade * material.valorUnitario`), centro de custo, data e usuário responsável.
- O filtro `termo` procura em nome do material e centro de custo dentro da lista já carregada.

Componentes e dependências
--------------------------
- `src/pages/EntradasPage.jsx`
- `src/components/TablePagination.jsx`
- `src/services/api.js` / `src/services/localApi.js`
- `src/styles/EntradasPage.css` (quando aplicável)

Relacionamentos
---------------
- As entradas alimentam os cálculos de estoque (`docs/Estoque.txt`) e o dashboard principal (`docs/Dashboard.txt`).
- Registros servem de base para o termo de EPI ao calcular quantidades entregues versus disponíveis.
